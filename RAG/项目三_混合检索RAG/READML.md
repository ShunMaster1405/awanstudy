# 项目三：混合检索RAG系统

## 项目概述

本项目实现了一个混合检索RAG（Retrieval-Augmented Generation）系统，结合了密集检索和稀疏检索的优势，提供更准确、更全面的信息检索和答案生成能力。

## 系统架构

### 核心模块

1. **文档处理模块** (`document_processor.py`)
   - 加载和预处理文档
   - 文档分割和关键词提取
   - 支持中文和英文文本处理

2. **混合索引模块** (`hybrid_index.py`)
   - 密集向量索引：使用嵌入模型生成文档向量
   - 稀疏向量索引：基于BM25和TF-IDF的关键词索引
   - 支持索引的保存和加载

3. **混合检索器模块** (`hybrid_retriever.py`)
   - 执行混合检索（结合密集和稀疏检索）
   - 支持单独使用密集或稀疏检索
   - 提供检索结果分析和比较

4. **答案生成器模块** (`answer_generator.py`)
   - 使用DeepSeek LLM生成答案
   - 基于检索结果构建上下文
   - 答案质量评估

### 简化版本

由于网络连接问题，我们还提供了一个简化版本 (`main_simple.py`)，它：
- 不使用外部嵌入模型
- 基于关键词匹配和随机向量进行检索
- 完全本地运行，无需网络连接

## 技术特点

### 混合检索策略

1. **密集检索**：
   - 使用向量相似度匹配
   - 捕捉语义相似性
   - 适合处理复杂查询和语义匹配

2. **稀疏检索**：
   - 基于关键词匹配（BM25 + TF-IDF）
   - 捕捉精确的词汇匹配
   - 适合处理具体术语和精确查询

3. **混合融合**：
   - 使用加权平均或RRF（Reciprocal Rank Fusion）融合两种检索结果
   - 平衡语义匹配和词汇匹配的优势

### 优势

1. **提高召回率**：结合两种检索方法，覆盖更多相关文档
2. **提高准确率**：通过融合策略筛选最相关结果
3. **鲁棒性**：当一种方法失效时，另一种方法仍能工作
4. **灵活性**：可根据需求调整密集和稀疏检索的权重

## 使用方法

### 完整版本（需要网络连接）

```bash
# 安装依赖
pip install -r requirements.txt

# 运行系统
python main_actual.py --question "你的问题"

# 交互模式
python main_actual.py --interactive

# 演示模式
python main_actual.py
```

### 简化版本（无需网络连接）

```bash
# 直接提问
python main_simple.py --question "人工智能是什么"

# 演示模式
python main_simple.py
```

## 配置说明

### 环境变量（完整版本需要）

在 `.env` 文件中配置：

```bash
# DeepSeek API配置
DEEPSEEK_API_KEY=your_api_key_here
DEEPSEEK_API_BASE=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-chat

# 嵌入模型配置
EMBEDDING_MODEL=BAAI/bge-m3

# 检索配置
SPARSE_TOP_K=5
DENSE_TOP_K=5
RRF_K=60

# 路径配置
DATA_PATH=./data/documents
VECTOR_STORE_PATH=./vector_index
```

### 参数说明

- `SPARSE_TOP_K`：稀疏检索返回的文档数量
- `DENSE_TOP_K`：密集检索返回的文档数量
- `RRF_K`：RRF融合参数
- `CHUNK_SIZE`：文档分割大小
- `CHUNK_OVERLAP`：文档分割重叠大小

## 扩展性

### 添加新的检索方法

1. 在 `hybrid_index.py` 中添加新的检索方法
2. 在 `hybrid_retriever.py` 中集成新方法
3. 调整融合策略以包含新方法

### 支持新的文档格式

1. 在 `document_processor.py` 中添加新的文档解析器
2. 支持PDF、Word、HTML等格式

### 优化检索性能

1. 使用更高效的向量数据库（如FAISS、Chroma）
2. 实现索引压缩和量化
3. 添加缓存机制

## 故障排除

### 常见问题

1. **网络连接问题**：
   - 使用简化版本 (`main_simple.py`)
   - 配置代理或使用本地模型

2. **内存不足**：
   - 减少 `CHUNK_SIZE`
   - 使用更小的嵌入模型
   - 启用索引分片

3. **检索结果不准确**：
   - 调整密集和稀疏检索的权重
   - 优化文档预处理
   - 增加训练数据

### 调试模式

系统提供了详细的日志输出，可以通过调整日志级别来调试：
- 设置 `LOG_LEVEL=DEBUG` 查看详细日志
- 检查检索过程中的中间结果
